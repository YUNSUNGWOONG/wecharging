# Uncertainty Inventory — Phase 0

이 문서는 **하드웨어 발주 전에 깨질 수 있는 가정들의 목록**입니다. 살아있는
문서로 유지하며, 실험 결과가 나올 때마다 업데이트합니다.

## 사용법

1. **수집 단계**: 팀 워크샵 1~2시간으로 가정을 30~50개 일단 다 적습니다.
   "그건 당연하잖아" 라고 느껴지는 것도 적습니다 (그 당연함이 깨지는 게
   가장 비쌉니다).
2. **점수 단계**: 각 항목에 Impact (1~5) × Probability (1~5) 를 매깁니다.
   - Impact 5: 깨지면 프로젝트 사망
   - Impact 1: 깨지면 약간 불편
   - Probability 5: 가만히 두면 거의 확실히 깨짐
   - Probability 1: 거의 깨질 일 없음
3. **정렬 단계**: Impact × Probability 점수 내림차순으로 정렬.
   상위 5~10개에 대해 `EXPERIMENT_PROTOCOLS.md` 작성.
4. **검증 단계**: 실험 결과를 Status 열에 기록. Kill criteria 위반 시
   즉시 팀 회의.

## Status 코드

- `OPEN` — 아직 측정 안 됨 (기본값)
- `IN_PROGRESS` — 실험 진행 중
- `VERIFIED` — 가정 확인됨, 안전
- `PARTIAL` — 일부 조건에서만 성립, 다운스코프 검토
- `BROKEN` — 가정 깨짐, **Kill criteria 발동**
- `MITIGATED` — 가정은 깨졌지만 우회 설계로 해결
