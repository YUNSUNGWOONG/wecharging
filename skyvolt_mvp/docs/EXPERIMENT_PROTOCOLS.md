# Experiment Protocols — Phase 0 Risk Reduction

`UNCERTAINTY_INVENTORY.md` 의 상위 14개 가정을 검증하기 위한 **실험 프로토콜**.

각 실험은 다음 형식을 따릅니다:

- **목적**: 어느 가정을 검증하는가
- **가설**: 무엇이 참이라고 보고 있는가
- **장비**: 필요한 것 (가급적 100만원 이내)
- **절차**: 누구든 재현할 수 있도록 단계별로
- **데이터 기록**: 어떤 형식으로
- **분석**: 어떤 통계 / 시각화
- **판정 기준**: 어떤 결과면 PASS / FAIL / 다운스코프
- **소요 시간**

---

## EXP-01 — RFID 콘크리트 환경 검출률 (가정 S3)

**우선순위**: 최상위. 깨지면 위치 추정 방식 자체를 다시 설계해야 함.

**목적**: 지하주차장 철근 콘크리트 천장 환경에서 IQT1-FP-R4-V1 리더의
검출 신뢰성을 측정.

**가설**: 검출률 ≥ 95%, 검출 거리 분산 ≤ 자유공간의 1.5배.

**장비**:
- IQT1-FP-R4-V1 RFID 리더 1개 (~30만원)
- 호환 태그 20개 (~5만원)
- USB-시리얼 어댑터 + 노트북
- 줄자, 마커, 삼각대

**대상 사이트** (5곳, 다양성 확보):
- 강남 오피스 빌딩 지하 1층
- 마포 신축 아파트 지하 2층
- 판교 IT 빌딩 지하 1층
- 송도 노후 상가 지하 1층
- 어디든 노출 콘크리트 천장 vs 마감재 천장 비교

**절차**:
1. 리더를 삼각대에 천장 방향으로 고정 (지면에서 1m).
2. 태그를 정사각형 격자 (50mm 간격, 11×11 = 121 위치) 로 천장에 임시 부착.
3. 각 위치에서 30초간 검출 시도, 검출 횟수 / RSSI 기록.
4. 같은 격자를 자유공간(실내 책상 위) 에서 측정해 baseline 으로 사용.
5. 5개 사이트 모두에서 반복.

**데이터 기록**:

```
site_id, tag_position_x_mm, tag_position_y_mm, n_detections_in_30s, mean_rssi_dBm, std_rssi_dBm
```

CSV 1개 파일로 누적, 사이트당 ~600 row.

**분석**:
- 사이트별 검출률 히스토그램
- RSSI 분포 자유공간 대비
- 콘 형상 (검출률 ≥ 80% 영역) 의 반경 계측
- 사이트 간 일관성 (분산)

**판정 기준**:
- **PASS**: 모든 사이트에서 50mm 콘 반경 ≥ 80% 검출률
- **PARTIAL**: 일부 사이트만 통과 → 추가 센서 (홀/QR) 검토
- **FAIL**: 콘 반경 < 30mm 또는 검출률 < 70% → 위치 추정 방식 재설계

**소요 시간**: 1주 (출장 3일 + 분석 2일)

**비용**: 약 40만원

## EXP-02 — 광전센서 트리거 정밀도 @ 1.5m/s (가정 S2)

**우선순위**: 최상위. ±3mm 도킹 클레임의 직접적 검증.

**목적**: EE-SPY401-2 광전센서가 빠른 속도에서 위치를 ±1mm 안에 트리거
하는지 측정.

**가설**: 1.5m/s에서 트리거 위치 표준편차 ≤ 1mm.

**장비**:
- EE-SPY401-2 광전센서 1개 (~15만원)
- 1m 리니어 가이드 + NEMA17 스텝모터 (~30만원)
- Arduino + 마이크로초 타이머
- 도킹 카드 모형 (3D 프린트, 두께 5mm 차광판)

**절차**:
1. 리니어 가이드 위에 도킹 카드 고정.
2. 광전센서를 슬라이더에 장착, 0.5/1.0/1.2/1.5/1.8 m/s 속도로 통과.
3. 센서 트리거 신호의 rising edge 시각을 마이크로초 단위 기록.
4. 모터 엔코더의 위치를 동시 기록.
5. 각 속도에서 100회 반복.

**데이터 기록**:

```
target_speed_mps, actual_speed_mps, trigger_position_mm, t_us
```

500 row.

**분석**:
- 속도별 트리거 위치 분포 (평균 + 표준편차)
- 속도가 정밀도에 미치는 영향 (선형 회귀)
- p99 트리거 오차

**판정 기준**:
- **PASS**: 1.5m/s에서 표준편차 ≤ 1mm, p99 ≤ 3mm
- **PARTIAL**: 1.0m/s에서만 통과 → 시스템 속도 다운스코프 (1.0m/s)
- **FAIL**: 1.0m/s에서도 표준편차 > 5mm → 추가 센서 (광학 인코더 / LiDAR)

**소요 시간**: 1주 (셋업 3일 + 측정 2일 + 분석 2일)

**비용**: 약 50만원

## EXP-03 — 한국 지하주차장 환경 실사 (가정 E1, E2)

**우선순위**: 최상위. 시장 진입성의 직접 검증.

**목적**: 한국 도심 지하주차장의 천장 높이, 매달 공간, RF 환경을 실측.

**가설**:
- 70% 이상 시설이 천장 ≥ 2.5m
- 60% 이상 시설에서 천장 매달 공간 확보 가능

**장비**:
- 레이저 거리계 (Bosch GLM 50, ~7만원)
- 카메라 1대 (스마트폰 가능)
- 측정 양식 (아래)

**대상**: 5개 지역 × 4곳 = 20곳
- 강남구 (오피스 밀집), 마포구 (구도심+신축 혼재),
  성동구 (재건축 다수), 판교 (신축), 송도 (계획도시)
- 각 지역에서 오피스 / 아파트 / 상가 / 호텔 1곳씩

**절차**:
1. 시설 입구에서 운영자 / 관리실에 측정 허가 (촬영 동의).
2. 입구에서 가장 가까운 충전 가능 구역으로 이동.
3. 다음을 측정:
   - 천장 높이 (3개 지점 평균)
   - 천장 마감재 종류 (콘크리트 노출 / 석고보드 / 금속패널)
   - 배관/조명 매단 가장 낮은 지점 높이
   - 기둥 간격
   - Wi-Fi / 4G 신호 세기 (스마트폰 앱)
   - 주차된 EV 수 / 충전기 수
4. 사진 5장 이상.

**데이터 기록**: Google Sheets / Excel 한 줄 = 한 시설:

```
site_id, region, type, ceiling_h_avg_m, ceiling_min_obstruction_m,
finish_type, column_spacing_m, wifi_dbm, lte_dbm,
n_ev_observed, n_chargers_installed, photos
```

**분석**:
- 천장 높이 히스토그램 (전체, 시설 유형별)
- 매달 공간 확보율 (= ceiling_min_obstruction_m ≥ 2.5)
- 시설 유형 × 지역 cross-tab

**판정 기준**:
- **PASS**: 천장 ≥ 2.5m 비율 ≥ 70%, 매달 공간 확보 ≥ 60%
- **PARTIAL**: 한 가지만 만족 → 타깃 시장 좁히기 (예: 신축 오피스)
- **FAIL**: 둘 다 < 50% → 시스템 형식 재고 (저천장형)

**소요 시간**: 2주 (현장 1주 + 분석 3일)

**비용**: 약 10만원 + 출장비

## EXP-04 — 차량 충전구 위치 분포 (가정 E3)

**우선순위**: 상위. 매니퓰레이터 작업 영역의 입력값.

**목적**: 일반 EV 사용자가 주차장에 입차했을 때 충전구의 실제 위치
분포 (3D 좌표) 측정.

**가설**: 차량 모델별로 충전구 위치 표준편차 ≤ 30cm.

**장비**:
- USB 카메라 1대 (~10만원)
- 라벨링 소프트웨어 (CVAT / Roboflow 무료 플랜)
- 노트북

**절차**:
1. 협력 지하주차장 1곳에 카메라 고정 (1주일 기록).
2. 약 30~50대 EV 입차 영상 수집.
3. 각 영상에서 차량 모델, 주차 후 충전구 위치 (지면 기준 x, y, z)
   수동 라벨링.
4. 차량 모델별 평균 + 분산 계산.

**데이터 기록**:

```
timestamp, vehicle_model, parking_slot_id,
charge_port_x_cm, charge_port_y_cm, charge_port_z_cm
```

50~100 row.

**분석**:
- 차량 모델별 충전구 평균 위치 (top 10 모델)
- 위치 분산 (모델 내, 모델 간)
- 차량 입차 위치 변동성과의 결합 영향

**판정 기준**:
- **PASS**: 모델 내 표준편차 ≤ 15cm, 모델 간 평균 차이 ≤ 50cm
- **PARTIAL**: 일부 모델 (예: 픽업) 만 큼 → 적용 차종 제한
- **FAIL**: 표준편차 > 50cm → 매니퓰레이터 자유도 추가, 비전 시스템 필수

**소요 시간**: 2주 (촬영 1주 + 라벨링/분석 1주)

**비용**: 약 20만원 + 협력 사이트 사용료

## EXP-05 — 트랙 열팽창 영향 (가정 M2)

**우선순위**: 상위. 한국 환경에서 사계절 운영 가능성 검증.

**목적**: 4060 알루미늄 프로파일 7m 길이가 -5℃~35℃ 변화 시 도킹 정확도에
미치는 영향 측정.

**가설**: 열팽창에 의한 도킹 위치 편차는 허용 마진 ±10mm 내.

**장비**:
- 4060 알루미늄 프로파일 1m 샘플 1개 (~3만원)
- 항온 챔버 또는 자연 환경 (옥상 24시간 측정)
- 온도 센서 + 변위 센서 (LVDT, ~30만원)
- 데이터 로거

**절차**:
1. 1m 프로파일을 한 끝 고정, 반대 끝에 LVDT 부착.
2. 온도와 변위를 24시간 동시 기록 (1초 간격).
3. 회귀 분석으로 열팽창 계수 측정 (이론값 23.1 μm/m·℃ 와 비교).
4. 측정값을 7m 트랙 + RFID 태그 위치에 외삽.
5. 시뮬레이터에 적용해 도킹 시뮬 100회 돌리고 위치 편차 측정.

**데이터 기록**:

```
t_s, temp_c, displacement_um
```

86,400 row (1초 × 24h).

**분석**:
- 측정 열팽창 계수 vs 이론값
- 가장 큰 일일 온도 변화 × 7m 트랙 → 예상 절대 편차
- 시뮬레이션 결과 (도킹 ±?mm)

**판정 기준**:
- **PASS**: 일일 변화 도킹 영향 ≤ 5mm
- **PARTIAL**: 영향 5~15mm → 트랙 분할 조인트 추가, 보정 알고리즘
- **FAIL**: 영향 > 15mm → 트랙 재료 변경 (스틸?), 또는 RFID 태그 재캘리브레이션 자동화

**소요 시간**: 2주

**비용**: 약 35만원

## EXP-06 — 다중 RFID 리더 간섭 (가정 D3)

**우선순위**: 상위. 다중 로봇 운영 가능성.

**목적**: 동일 주파수 RFID 리더 2~5대가 가까이 있을 때 검출률 영향 측정.

**가설**: 1m 이상 떨어진 리더 간 간섭은 무시 가능.

**장비**: EXP-01 의 리더 + 추가 리더 4대 (총 ~120만원)

**절차**:
1. 책상 위에 리더 5대를 1m 간격 직선 배치, 모두 동시 가동.
2. 태그 1개를 각 리더 위치에 차례로 두고 검출률 측정.
3. 리더 간격을 0.5m / 1.0m / 1.5m / 2.0m 로 변화시켜 반복.
4. 베이스라인 (1대만 가동) 과 비교.

**판정 기준**:
- **PASS**: 1m 간격에서 검출률 저하 ≤ 5%
- **PARTIAL**: 1.5m 간격 필요 → 트랙 설계 수정 (RFID 태그 간격)
- **FAIL**: 2m 간격에서도 저하 > 10% → 시간 분할 다중화 또는 다른 주파수 채택

**소요 시간**: 1주

**비용**: 약 120만원 (리더 추가 4대)

## EXP-07 — 시장 수요 인터뷰 (가정 B1, B3)

**우선순위**: 최상위 (사업 측면). RE는 데이터를 받기만 하면 됨.

**목적**: (1) 시간당 충전 수요, (2) 주차장 운영자 수용성을 정성/정량 측정.

**담당**: PM / 사업개발 팀.

**가설**:
- 도심 EV 충전 가능 주차장에서 시간당 요청 ≥ 5회
- 주차장 운영자 50% 이상이 천장 설치 동의 (조건부 포함)

**절차**:
1. EV 사용자 30명 인터뷰 (충전 빈도, 대기 시간 허용도, 가격 민감도)
2. 주차장 운영자 10곳 인터뷰 (운영 모델, 설치 허용 조건, 임대료 기대)
3. 기존 충전소 5곳 데이터 협조 요청 (시간대별 가동률)

**판정 기준**:
- **PASS**: 두 가설 모두 충족
- **PARTIAL**: 한 가지만 → 비즈니스 모델 재설계
- **FAIL**: 둘 다 미충족 → 프로젝트 일시중지

**소요 시간**: 4주
